### 基本用法
```bash

/review

# 根据提示选择报告类型（1-7）
# 选择时间周期
# 选择是否自动填充数据
```

bash ~/.claude/skills/review/scripts/detect-repeated-changes.sh "2026-04-23"

## 核心特性

### 1. 数据驱动
所有报告必须基于以下数据源：
- Git 提交记录
- Session 交互历史
- Traces 执行日志
- Tool 调用记录

### 2. 三级分类
- ✅ 高效动作（保持并推广）
- ⚠️ 可优化动作（改进流程）
- 🔴 禁止动作（立即杜绝）

### 3. 反复修改检测
自动检测同一问题被修改 ≥3 次，标注效率损失。

### 4. Token 分析
自动识别 Token 浪费，计算优化潜力和成本。

### 5. 强制行动
每次复盘必须输出具体改进动作，自动转化为下一周期计划。

## 常见问题

### Q1: 为什么要重构？
**A**: 旧版 SKILL.md 1730 行，远超官方建议的 500 行。这导致：
- 加载慢（~6900 tokens）
- 维护难（所有内容混在一起）
- 扩展难（新增场景要改主文件）

### Q2: 重构后功能有变化吗？
**A**: 没有。功能完全一致，只是文件组织方式变了。

### Q3: 模板/脚本/参考文档还没创建完？
**A**: 是的。优先创建了最重要的日报模板和 SKILL.md。其他文件会逐步补充（参考"待补充内容"部分）。

### Q4: 如果我需要某个模板怎么办？
**A**: 可以从旧版 SKILL.md.backup 中提取对应章节，或者等待后续补充。
